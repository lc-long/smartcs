# 复杂多 Agent 协作测试场景

## 场景 1：退款+账单核查

**用户问题**：
```
我是客户C001，我的智能手表Pro屏幕闪烁，我想退货退款。另外，我上个月的账单好像多扣了钱，帮我一起查一下。
```

**预期执行流程**：
1. Supervisor 分析 → 复杂任务（涉及退款+账单）
2. 拆解子任务：
   - Task 1: 查询订单 ORD-20260402-001 的退款资格 (Refund Agent)
   - Task 2: 查询账单是否有异常 (Billing Agent)
3. 并行执行 Task 1 和 Task 2
4. 汇总结果 → 综合回复

**验证点**：
- [ ] Supervisor 正确识别为复杂任务
- [ ] 两个 Agent 并行执行
- [ ] 最终回复包含退款和账单两部分信息

## 场景 2：技术问题+退款

**用户问题**：
```
我的智能手表Pro屏幕闪烁，已经影响使用了。我想了解一下这是什么问题，如果解决不了我想退款。
```

**预期执行流程**：
1. Supervisor 分析 → 复杂任务（技术+退款）
2. 拆解子任务：
   - Task 1: 搜索知识库，查找屏幕闪烁的解决方案 (Technical Agent)
   - Task 2: 查询退款资格 (Refund Agent)
3. 串行执行：先 Task 1，如果无法解决再 Task 2
4. 汇总结果 → 综合回复

**验证点**：
- [ ] Supervisor 正确识别为复杂任务
- [ ] 先尝试技术解决，再考虑退款
- [ ] 最终回复包含技术建议和退款选项

---

## 场景 3：多订单查询+对比

**用户问题**：
```
帮我查一下我最近的两个订单，对比一下金额和商品，看看哪个更划算。
```

**预期执行流程**：
1. Supervisor 分析 → 复杂任务（多订单查询+分析）
2. 拆解子任务：
   - Task 1: 查询订单列表 (General Agent)
   - Task 2: 查询商品详情 (Technical Agent)
3. 并行执行
4. 汇总结果 → 对比分析

**验证点**：
- [ ] Supervisor 正确识别为复杂任务
- [ ] 获取到订单和商品信息
- [ ] 最终回复包含对比分析

## 场景 4：退款+客服转接

**用户问题**：
```
我想退款，订单号是ORD-20260402-001。另外，我之前有个技术工单一直没解决，想转人工客服。
```

**预期执行流程**：
1. Supervisor 分析 → 复杂任务（退款+工单+人工）
2. 拆解子任务：
   - Task 1: 检查退款资格 (Refund Agent)
   - Task 2: 查询工单状态 (Technical Agent)
   - Task 3: 准备人工转接
3. 并行执行 Task 1 和 Task 2
4. 汇总结果 → 包含退款信息和工单状态，提示人工转接

**验证点**：
- [ ] Supervisor 正确识别为复杂任务
- [ ] 同时处理退款和工单查询
- [ ] 最终回复包含退款信息、工单状态和人工转接提示

---

## 场景 5：账单异常+退款申请

**用户问题**：
```
我发现订单ORD-20260401-001的账单金额是3298元，但商品总价应该是3297元，多扣了1块钱。帮我查一下，如果确实多扣了，我想申请退款。
```

**预期执行流程**：
1. Supervisor 分析 → 复杂任务（账单核查+退款）
2. 拆解子任务：
   - Task 1: 查询订单详情和账单 (Billing Agent)
   - Task 2: 计算商品总价对比账单金额
   - Task 3: 如果确实多扣，检查退款资格
3. 串行执行：先 Task 1 → Task 2 → Task 3
4. 汇总结果 → 综合回复

**验证点**：
- [ ] Supervisor 正确识别为复杂任务
- [ ] 能够发现账单异常（多扣1块钱）
- [ ] 最终回复包含账单分析和退款建议

## 测试优先级

| 场景 | 优先级 | 测试重点 |
|------|--------|----------|
| 场景 1 | 高 | 多 Agent 并行执行 |
| 场景 2 | 高 | 串行执行+条件判断 |
| 场景 3 | 中 | 数据聚合+分析 |
| 场景 4 | 中 | 多任务+人工转接 |
| 场景 5 | 高 | 复杂逻辑+条件退款 |
