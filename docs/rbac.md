# SmartCS 角色权限设计文档

## 1. 角色定义

系统定义三种角色，按权限从高到低排列：

| 角色 | 代码 | 说明 | 典型用户 |
|------|------|------|----------|
| 管理员 | `admin` | 系统管理者，拥有全部权限 | 运营经理、系统管理员 |
| 客服 | `agent` | 客服人员，处理客户问题 | 客服专员、客服主管 |
| 用户 | `viewer` | 注册用户，查看自己的数据 | 普通消费者 |

## 3. 数据可见性规则

### 3.1 订单数据
- **管理员**：可见所有订单
- **客服**：可见自己处理的订单 + 分配给自己的订单
- **用户**：仅可见自己的订单

### 3.2 对话数据
- **管理员**：可见所有对话记录
- **客服**：可见自己参与的对话（人工接管过的）
- **用户**：仅可见自己的对话

### 3.3 退款数据
- **管理员**：可见所有退款记录
- **客服**：可见待审批的退款 + 自己处理的退款
- **用户**：仅可见自己的退款记录

### 3.4 工单数据
- **管理员**：可见所有工单
- **客服**：可见分配给自己的工单 + 未分配的工单
- **用户**：仅可见自己创建的工单

## 4. 操作限制规则

### 4.1 金额限制
| 操作 | 客服限制 | 管理员 |
|------|----------|--------|
| 退款审批 | ≤¥500 | 无限制 |
| 订单取消 | ≤¥5000 | 无限制 |
| 创建优惠 | ❌ | 无限制 |

### 4.2 时间限制
| 操作 | 限制 |
|------|------|
| 退款申请 | 订单完成后 30 天内 |
| 工单关闭 | 解决后 7 天内自动关闭 |
| 对话查看 | 保留最近 90 天 |

### 4.3 频率限制
| 操作 | 限制 |
|------|------|
| API 调用 | 100 次/分钟（普通用户） |
| API 调用 | 1000 次/分钟（客服） |
| API 调用 | 无限制（管理员） |
| 退款申请 | 每个订单最多 3 次 |

## 5. 前端权限控制

### 5.1 路由权限
```
/login          → 无需登录
/               → 所有角色
/dashboard      → 所有角色（数据范围不同）
/products       → 所有角色（只读）
/orders         → 所有角色（操作权限不同）
/customers      → admin + agent
/refunds        → admin + agent
/knowledge      → 所有角色（编辑权限不同）
/admin          → 仅 admin
```

### 5.2 组件权限
- 按钮根据角色显示/隐藏
- 表单字段根据角色启用/禁用
- 数据列表根据角色过滤

### 5.3 导航权限
- 用户看不到无权访问的菜单项
- 侧边栏根据角色动态显示

---

## 6. 后端权限控制

### 6.1 API 级别
- 使用 JWT Token 验证身份
- 使用 `require_role` 依赖检查角色
- 每个 API 端点明确声明所需角色

### 6.2 数据级别
- 查询时根据角色过滤数据
- 用户只能操作自己的数据
- 客服只能操作分配给自己的数据

### 6.3 工具级别
- Agent 工具调用时传入 customer_id
- 工具内部检查数据归属
- 防止越权访问
